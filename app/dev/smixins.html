Brief Documentation for border-radius Mixins

  • .border-radius(10px 0 0 0 );
    • expects four non-string arguments;
    • Similar to border-radius : 10px 0 0 0;
      but prints the result in long-hand for older web-kit support.
  • .border-radius("100px",10% , 20% , 30% , 40%);
    • first argument passed as string
    • expects four arguments after
    • sets all the corners to 100px and each value after that sets the ellipse radius to the value clockwise from top-left to bottom-left.
  • .border-radius(20px);
    • is equivalent to the normal border-radius that sets all the corners to 20px; output is : border-radius : 20px;
  • .border-radius(20px 5%);
    • two arguments only separated by a space
    • sets all the corners to 20px and then the ellipse radius to 5% for each corner.
  • .border-radius("2px", "50px");
    • Expects only two arguments as strings.
    • .border-radius("2px", "50px"); is equivalent to border-radius : 2px 50px; But the difference is that the output is printed long-hand for maximum compatibility. Notice that both arguments are passed in as strings to avoid mixin overlap and conflict.
  • .border-radius("50%","60%",10%,20%,30%,40%);
    • The first two arguments are passed as strings
    • Expects fours arguments after the first two.
    • sets top-left and bottom-right to 50% and sets top-right and bottom-left to 60% in long-hand form. Then starting from top-left corner and moving clockwise, it sets the ellipse radius to 10%,20%,30% and 40%. Notice that the first and second arguments are passed as strings and all the four values are specified.
  • .border-radius("10px","40px",50%);
    • is equivalent to: .border-radius("10px","40px",50%,50%,50%,50%);
  • ._border-radius(10px,"top-left");
    • The first argument is a value, and then the rest are the corners explicitly passed as strings, in no particular order.
    • equivalent to : .border-radius(10px 0 0 0);
    • Note the underscore in the mixin definition. It makes it identifiable as a helper mixin. Directly uses the implementation for the border-radius helper.
  • ._border-radius(20%,"top-left","bottom-left");
    • The first argument is a value, and then the rest are the corners explicitly passed as strings, in no particular order.
    • Equivalent to .border-radius(20% 0 0 20%);
    • Note the underscore in the mixin definition. It makes it identifiable as a helper mixin. Directly uses the implementation for the border-radius helper.

Brief Documentation for box-shadow Mixins

  • .box-shadow(inner/outer, left/right/top/bottom, [@x , @y , @blur , @speed, @color ])
    • @x , @y , @blur , @speed, @color are optional parameters
    • The first and the second arguemnts are required and should be passed in as non-string values.
    • Example:
      • .box-shadow(inner, left, 1px, 2px, 3px, 4px, #000);
      • The first argument is required. It is a non-string parameter. The possible options are either inner or outer.
      • The second argument is required. It is a non-string parameter than can be either left, right, top or bottom.
      • The rest of the arguments are optional.
      • In the example above, 1px is the horizontal displacement, 2px is the vertical displacement, 3px is the amount of blur, 4px is the speed, and the last argument (#000) is the color.
    • The arguments can also be used explicitly in no particular order. But the first two arguments should be provided and the order is important.
      • Examples :
        • .box-shadow(inner, left, @blur:10px, @color:#fff);
        • .box-shadow(outer, right, @color : rgba(0,0,0,0.6), @y:10px, @speed : 1px, @blur:10px);
  • .box-shadow("arguments");
    • The mixin is very general, and takes only one argument. It is important to notice that the single argument should be padded as a string. It is essentially equivalent to the standard CSS3 specification.
    • Example:
      • .box-shadow("0 0 2px 0 #000, inset 0 0 1px 0 #000, 1px 2px 3px 4px rgba(0,0,0,0.4)");
      • Although the mixin is identical to the standard CSS3 specification, it has one advantage. And that is that it can be used in combination with the other box-shadow mixins. For instance:

        .multiple-shadows{
          .box-shadow(inner,left);
          .box-shadow(outer,top);
          .box-shadow("0 0 2px 0 #000, inset 0 0 1px 0 #000, 1px 2px 3px 4px rgba(0,0,0,0.4)");
        }

        This will result in the following output:

        .multiple-shadows{
        box-shadow: inset 3px 0 8px -3px #000000, 0 -3px 4px -1px #000000, 0 0 2px 0 #000, inset 0 0 1px 0 #000, 1px 2px 3px 4px rgba(0,0,0,0.4); }

  • .box-shadow-well([@blur, @speed, @color]);
    • Used for making well regions. It accepts three optional parameters: @blur, @speed and @color.
    • As with any mixin with default values, the parameters can be passed in explicitly.
    • Example:

      .example1{
        .box-shadow-well(@color : #000 , @blur : 20px);
      }

Brief Documentation for font-size Mixins

  • font-size maker:

    • @_base-fs : 13;
    @fs-lower-bound : 0;
    @fs-upper-bound : 20;
    @fs-steps : 2;

    .make-fs(@fs-upper-bound);
    .make-fs( @i: @fs-lower-bound ; @j : 1) when not(@i=0) {
       @v : (@i)/(1/((@_base-fs)))/100;
       .fs-@{j}{
          font-size : ~"@{v}em";
       }
       .make-fs( (@i - @fs-steps), (@j + 1));
    }

  • @_base-fs : sets the base font size from which other font size values are calculated.
  • @fs-lower-boud : sets the initial value of the font-size maker loop.
  • @fs-upper-bound : sets the last value of the font-size maker loop.
  • @fs-steps : sets the distances between each font size value.
